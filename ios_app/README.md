# Lingbot Capture (iOS, Tier 1 LiDAR)

Native SwiftUI app that captures an ARKit + LiDAR session and uploads the
resulting bundle to the lingbot-map backend.

This is the **phone-advantage** capture path: ARKit gives us per-frame camera
poses + intrinsics, the LiDAR sensor gives us metric depth, and the server
skips lingbot-map's depth/pose inference entirely. Result: faster server-side
processing (~2 min vs ~10–15 min) and substantially better geometry than the
raw-video path can produce.

The web upload at the same backend (and the parked Flutter app at
`attic/phone_app_tier3/`) remain as the Tier 3 fallback for non-ARKit devices.

## What the app captures

At ~10 fps while you record, each frame writes:

| File | Format | Source |
|---|---|---|
| `frames/000000.jpg` | sRGB JPEG, native AR resolution | `ARFrame.capturedImage` |
| `poses/000000.json` | intrinsics 3x3 + transform_c2w 4x4 | `ARFrame.camera.intrinsics` / `.transform` |
| `depth/000000.bin` | float32 LE meters, 192×256 (or device-native) | `ARFrame.sceneDepth.depthMap` |
| `depth/000000.conf` | uint8 ARConfidence (0/1/2) | `ARFrame.sceneDepth.confidenceMap` |
| `manifest.json` | tier, frame count, resolutions, device id | written on STOP |

On STOP it zips the session directory and posts it to
`POST /api/runs/bundle` on the deployed Modal backend
(`https://arielpollack--lingbot-map-web-fastapi-app.modal.run` — set in
[Sources/Backend.swift](Sources/Backend.swift)).

## Build + run on a physical device

LiDAR + ARKit don't work in the simulator — you need an iPhone Pro / Pro Max
(12 series or newer) or iPad Pro.

```sh
cd ios_app
xcodegen generate
open LingbotCapture.xcodeproj
```

In Xcode:

1. **Project → LingbotCapture target → Signing & Capabilities** —
   - Set **Team** to your personal Apple ID team.
   - Change **Bundle Identifier** to something globally unique
     (`com.lingbot.capture.<yourname>`).
2. Plug in the iPhone, select it as the run destination.
3. ▶︎ **Run**.

The first launch on a new dev cert will fail with "untrusted developer".
On the phone: **Settings → General → VPN & Device Management → trust your
dev profile**, then re-run.

`Info.plist` is generated by xcodegen from `project.yml`. If you change the
permissions strings or capabilities, edit `project.yml` and re-run
`xcodegen generate`.

## Backend pieces

- [poc/app/main.py](../poc/app/main.py) — `POST /api/runs/bundle` accepts
  the zipped capture as a single multipart `bundle` field.
- [poc/worker/pipeline.py](../poc/worker/pipeline.py) — detects
  `input_bundle_key` in the payload and routes to the LiDAR pipeline.
- [poc/worker/lidar_pipeline.py](../poc/worker/lidar_pipeline.py) — extracts
  the bundle, converts ARKit GL camera convention → OpenCV (the `gl_to_cv`
  diagonal flip), builds the `prepared` dict, then reuses the existing
  mesh / texture / 3DGS code unchanged.

## Redeploying the backend

```sh
modal deploy poc/app/modal_web.py     # FastAPI front
modal deploy poc/worker/modal_app.py   # GPU worker (lidar_pipeline.py)
```

Both deploys take ~3 s because they only refresh the local code mount;
nothing in the Docker image needs to rebuild.
